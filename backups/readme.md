(Source backups via amalgamated source file bundles.)

The .ma files each represent one project, and are an amalgamation of its source files.

mm8.ma is my latest systems language compiler, version 8. It was generated by the compiler itself.

These are readable (and compilable) source files, but some notes if perusing:

* All modules are combined into one
* Hard tabs are used that should displayed as four spaces
* Syntax is case-insensitive. (Mostly lower case except for temporary code or for highlighting)
* Text after "!" is comments

If the compiler is available, eg. mm8.exe, then a .ma file can be expanded into its discrete files. For example:
````
  mm8 -decons mm8.ma
````
This writes the files into src\\\*.\*. It can also be done with a script as this text format is not that complicated (see contents).
